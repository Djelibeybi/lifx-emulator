## Summary

Review and modernise the LIFX Emulator web UI to fix code quality issues identified in a design review, improve the visualizer's representation of physical devices, and bring the overall aesthetic up to modern standards.

## Context

The frontend is a SvelteKit 2 + Svelte 5 SPA that provides real-time monitoring and management of emulated LIFX devices. A comprehensive design review identified issues across visual design, usability, code quality, and performance. The visualizer tab — showing live color state of emulated bulbs, multizone strips, and matrix/tile devices — is the primary view and needs the most attention.

## Motivation

The current UI is functional but visually dated, wastes space, and has code quality issues including duplicated logic across components, missing accessibility attributes, undefined CSS variables, and an incorrect color conversion pipeline that ignored kelvin color temperature (rendering all white lights as flat grey).

## Technical Notes

- Color conversion pipeline: HSBK → blend kelvin with HSV hue → linear sRGB → Oklab matrix → Oklch polar → CSS `oklch()` output
- Ambilight glow uses `inset` box-shadow with edge-sampled colors averaged in linear RGB space
- Tile sizing uses `PX_PER_TILE_COL = 20` constant (8-column tile = 160px, 16-column = 320px)
